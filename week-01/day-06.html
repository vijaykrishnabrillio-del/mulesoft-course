🎯 Practical Example: Modular API

Scenario: Customer Management API

<!-- Main entry point -->
<flow name="customer-api-main">
    <http:listener 
        config-ref="HTTP_Listener_config" 
        path="/customer"/>
    
    <logger message="Request received"/>
    
    <!-- Call validation flow -->
    <flow-ref name="validate-request-subflow"/>
    
    <!-- Call processing flow -->
    <flow-ref name="process-customer-subflow"/>
    
    <!-- Call response builder flow -->
    <flow-ref name="build-response-subflow"/>
</flow>

<!-- Validation sub-flow -->
<sub-flow name="validate-request-subflow">
    <logger message="Validating request"/>
    <set-variable 
        variableName="isValid" 
        value="#[payload.email != null]"/>
    <logger message="#['Validation result: ' ++ vars.isValid]"/>
</sub-flow>

<!-- Processing sub-flow -->
<sub-flow name="process-customer-subflow">
    <logger message="Processing customer data"/>
    <set-variable 
        variableName="customerId" 
        value="#[randomInt(10000)]"/>
    <set-payload value="#[
        payload ++ {'customerId': vars.customerId}
    ]"/>
</sub-flow>

<!-- Response builder sub-flow -->
<sub-flow name="build-response-subflow">
    <logger message="Building response"/>
    <set-payload value="#[
        {
            'status': 'success',
            'data': payload
        }
    ]"/>
</sub-flow>

Test:

POST http://localhost:8081/customer
{
  "name": "Alice Johnson",
  "email": "alice@example.com"
}

Response:
{
  "status": "success",
  "data": {
    "name": "Alice Johnson",

}

When to Use Which:

Use Case Recommendation
Simple logic extraction Sub-flow
Shared across multiple flows Sub-flow or Private Flow
Database transaction Private Flow
Pure transformation Sub-flow

⚡ Async Scope

What is Async?

Async scope executes processors asynchronously (in parallel) without blocking the main flow.

Use Cases:

  • Sending notifications (don't wait)
  • Logging to external systems
  • Non-critical operations
  • Fire-and-forget patterns

Example: Send Email Notification Async

<flow name="order-processing-flow">
    <http:listener path="/order"/>
    
    <logger message="Processing order"/>
    
    <!-- Main processing (synchronous) -->
    <set-payload value="#[
        {
            'orderId': randomInt(10000),
            'status': 'confirmed'
        }
    ]"/>
    
    <set-variable variableName="orderId" value="#[payload.orderId]"/>
    
    <!-- Send notification asynchronously (non-blocking) -->
    <async doc:name="Send Notification Async">
        <logger message="#['Sending notification for order: ' ++ vars.orderId]"/>
        <!-- Simulate email sending -->
        <flow-ref name="send-email-subflow"/>
    </async>
    
    <!-- Response returns immediately without waiting for async -->
    <logger message="Returning response to client"/>
</flow>

<sub-flow name="send-email-subflow">
    <!-- Simulate slow email service -->
    <logger message="Email sending started"/>
    <set-payload value="#['Email sent for order: ' ++ vars.orderId]"/>
</sub-flow>

How it Works:

Timeline:
0ms:  Request received
1ms:  Order processing starts
2ms:  Async scope triggered (runs in background)
3ms:  Response sent to client ✓
...
100ms: Async completes (client already got response)

🛠️ Hands-On Exercise: Build Modular Application

Project: order-management-api

Requirements:

  1. Accept order via POST /order
  2. Validate order data (separate sub-flow)
  3. Calculate totals (separate sub-flow)
  4. Send confirmation email async
  5. Return order confirmation

    6. Implementation:

    <flow name="order-api-main">
    <http:listener path="/order" allowedMethods="POST"/>
    
    <logger message="Order received"/>
    <flow-ref name="validate-order-subflow"/>
    <flow-ref name="calculate-totals-subflow"/>
    
    <!-- Save order ID before async -->
    <set-variable variableName="orderId" value="#[payload.orderId]"/>
    
    <!-- Send email async (non-blocking) -->
    <async>
        <flow-ref name="send-confirmation-email"/>
    </async>
    
    <flow-ref name="format-response-subflow"/>
</flow>

<sub-flow name="validate-order-subflow">
    <logger message="Validating order"/>
    <!-- Add validation logic -->
    <set-variable 
        variableName="isValid" 
        value="#[payload.items != null and sizeOf(payload.items) > 0]"/>
</sub-flow>

<sub-flow name="calculate-totals-subflow">
    <logger message="Calculating totals"/>
    <set-variable 
        variableName="total" 
        value="#[sum(payload.items map $.price)]"/>
    <set-payload value="#[
        payload ++ {
            'orderId': 'ORD-' ++ randomInt(10000),
            'total': vars.total
        }
    ]"/>
</sub-flow>

<sub-flow name="send-confirmation-email">
    <logger message="#['Sending email for order: ' ++ vars.orderId]"/>
    <!-- Email sending logic here -->
</sub-flow>

<sub-flow name="format-response-subflow">
    <set-payload value="#[
        {
            'success': true,
            'orderId': payload.orderId,
            'total': payload.total,
            'message': 'Order confirmed'
        }
    ]"/>
</sub-flow>
