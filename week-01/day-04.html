Day 4: Your First Mule Application

📚 Learning Objectives

  • Build a complete "Hello World" REST API from scratch
  • Understand HTTP Listener configuration
  • Use Logger and Set Payload components
  • Run and test your Mule application
  • Debug using Console and Postman

Estimated Time: 3 hours

Theory: 20% | Practical: 80%

🚀 Step-by-Step Build

Step 1: Create New Project

  1. File → New → Mule Project
  2. Configuration:

    3. ```

    Project name: hello-world-api

    Runtime: Mule Server 4.x

    ```

    1. Click Finish

      2. Step 3: Build Endpoint 1 - Simple Hello World

      Drag Components to Canvas:

      1. From Mule Palette → HTTP, drag Listener to canvas
      2. From Mule Palette → Core, drag Set Payload after Listener
      3. From Mule Palette → Core, drag Logger before Set Payload

        4. Configure Flow:

        Flow Structure:

        [HTTP Listener] → [Logger] → [Set Payload]

        1. Configure HTTP Listener:

        Display Name: Receive Hello Request
Connector configuration: HTTP_Listener_config
Path: /hello
Allowed Methods: GET

        2. Configure Logger:

        Display Name: Log Request
Message: "Received request at /hello endpoint"
Level: INFO

        3. Configure Set Payload:

        Display Name: Set Hello Response
Value: "Hello, World!"

        Complete XML for Flow 1:

        <flow name="hello-world-flow">
    <http:listener 
        doc:name="Receive Hello Request" 
        config-ref="HTTP_Listener_config" 
        path="/hello"
        allowedMethods="GET"/>
    
    <logger 
        level="INFO" 
        doc:name="Log Request" 
        message="Received request at /hello endpoint"/>
    
    <set-payload 
        value="Hello, World!" 
        doc:name="Set Hello Response"/>
</flow>

        Step 5: Build Endpoint 2 - Personalized Greeting

        Add New Flow:

        Goal: GET /hello/John returns Hello, John!

        1. Create New Flow:

        • Drag new HTTP Listener to canvas (creates new flow automatically)

        2. Configure Components:

        HTTP Listener:

        Path: /hello/{name}
Allowed Methods: GET

        Logger:

        Message: #["Greeting request for: " ++ attributes.uriParams.name]

        Set Payload:

        Value: #["Hello, " ++ attributes.uriParams.name ++ "!"]

        Complete XML for Flow 2:

        <flow name="personalized-greeting-flow">
    <http:listener 
        doc:name="Receive Name Parameter" 
        config-ref="HTTP_Listener_config" 
        path="/hello/{name}"
        allowedMethods="GET"/>
    
    <logger 
        level="INFO" 
        doc:name="Log Name" 
        message="#['Greeting request for: ' ++ attributes.uriParams.name]"/>
    
    <set-payload 
        value="#['Hello, ' ++ attributes.uriParams.name ++ '!']" 
        doc:name="Set Personalized Greeting"/>
</flow>

        Understanding DataWeave Expressions:

        The #[ ] syntax denotes DataWeave expressions:

        #["Hello, " ++ attributes.uriParams.name ++ "!"]
│  │         │   │                          │
│  │         │   │                          └─ String concatenation
│  │         │   └─ URI parameter (from path)
│  │         └─ Concatenation operator
│  └─ String literal
└─ DataWeave expression delimiter
        • attributes.uriParams.name - Accesses the {name} from URL path
        • ++ - String concatenation operator
        • #[ ] - Wraps DataWeave expressions

        🚨 Troubleshooting Guide

        Issue 1: Port Already in Use

        Error: Address already in use: bind

        Solution:

        Option 1: Change port in HTTP Listener config to 8082
Option 2: Stop other application using port 8081

        Issue 2: Application Won't Start

        Error: Various startup errors

        Solution:

        1. Check Console for specific error
2. Project → Clean
3. Right-click project → Mule → Clear Application Data
4. Restart Studio

        Issue 3: 404 Not Found

        Error: Endpoint returns 404

        Solution:

        1. Verify exact path in Listener configuration
2. Check application is fully deployed (Console message)
3. Ensure HTTP method matches (GET vs POST)

        Issue 4: Empty Response

        Error: No response body

        Solution:

        1. Check Set Payload configuration
2. View Console for errors
3. Verify payload value expression
